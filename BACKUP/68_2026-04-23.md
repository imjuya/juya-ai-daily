## [OpenAI 发布开源隐私过滤模型，支持本地检测脱敏](https://openai.com/index/introducing-openai-privacy-filter/) `#4`
> **OpenAI** 发布了名为 **OpenAI Privacy Filter** 的开放权重模型。
>
> 该模型总参数量为 `1.5B`，激活参数为 `50M`。
>
> 它主要用于检测和脱敏文本中的个人身份信息。

**OpenAI** 发布了名为 **OpenAI Privacy Filter** 的开放权重模型，该模型采用 `Apache-2.0` 协议，旨在检测和脱敏文本中的个人身份信息 (`PII`)。

据官方博客描述，这是一个基于 `GPT-OSS` 架构的双向 token 分类模型，具备前沿的个人数据检测能力。它专为高吞吐量隐私工作流设计，能够对非结构化文本进行上下文感知的单次快速脱敏。

该模型采用了 `MoE` 架构，总参数量为 **1.5B**，激活参数为 **50M**，并支持 **128k** 上下文。由于体积小巧，该模型支持本地运行，甚至能够在浏览器中执行，从而确保敏感数据不会离开用户设备。

## 小米 MiMo 升级 Token 计划，调整计费及订阅模式 `#6`
> **Xiaomi MiMo** 宣布全面升级 `Token Plan`，上线了最新的 `MiMo-V2.5` 系列模型。
>
> 此举简化了 `Credits` 计费方式，并推出了夜间八折优惠。此外，官方还新增了连续包月和包年等订阅折扣。
>
> 同时，官方已重置已购用户的 `Credits` 额度。

**Xiaomi MiMo** 官方宣布对其 `Token Plan` 进行焕新升级，上线了最新的 **MiMo-V2.5** 系列模型并推出包括调整 `Credits` 计费倍率、新增夜间优惠以及丰富订阅折扣等多项优化措施。

在计费机制上，该计划取消了 `256k` 与 `1 M` 上下文窗口的 `Credit` 倍率差异及 `1 Token` 等于 `4 Credits` 的计费方式。此次调整将 `MiMo-V2.5` 和 **MiMo-V2.5-Pro** 的消耗速率分别设定为 `1x` 与 `2x`。

此外，此次升级引入了北京时间每日 `00:00` 至 `08:00` 的全模型夜间优惠 **8 折** 机制。

在订阅模式方面，官方新增了「连续包月」与「包年」选项。开通连续包月自动续费的老用户和新用户可分别享受一次性的次月 **7 折** 与 **77 折** 优惠，而包年订阅则提供全年 **88 折**，该折扣不再叠加其他首购或自动续费优惠。

与此同时，官方将已购买该套餐的所有用户进行 `Credits` 额度重置清零。

## [Zed 推出 Parallel Agents，支持多 Agent 并行运行](https://zed.dev/blog/parallel-agents) `#8`

**Zed** 编辑器在最新版本中推出了 **Parallel Agents** 功能，允许用户在同一个窗口内并行运行和编排多个 **Agent**。

## [Google 开源 DESIGN.md 规范](https://github.com/google-labs-code/design.md) `#9`
> **Google** 开源了其专门用于帮助 **AI** 编程智能体准确理解和执行项目设计的 `DESIGN.md` 规范。
>
> 该规范结合了机器可读的设计 token 与人类可读的文本说明，并同步推出了配套的命令行工具。

**Google** 旗下 **Stitch** 团队近期开源 `DESIGN.md` 规范，旨在为编程 Agent 建立视觉识别共享格式。

据官方称，该规范通过结合机器可读的 `设计 token` 与人类可读的设计原理，让 Agent 在跨工具和平台操作时无需再猜测设计意图。

该规范提供了一个配套的命令行工具，支持结构校验、无障碍对比度检查、版本差异对比以及向 `Tailwind` 或 `W3C DTCG` 标准的导出功能。

目前该格式仍处于 `alpha` 版本，官方表示规范正在积极开发中。
