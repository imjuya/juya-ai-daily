## [Anthropic 推出 Claude Design 工具](https://www.anthropic.com/news/claude-design-anthropic-labs) `#1`
> Anthropic推出了由 **Opus 4.7** 模型驱动的实验性视觉创作产品 **Claude Design**。
>
> **Claude** 订阅用户现在只需通过对话，就能快速生成并微调符合企业品牌风格的交互原型和幻灯片。
>
> 该工具不仅与 `Canva` 实现了无缝对接，还能将最终设计一键打包交接给 `Claude Code` 进行后续代码开发。

Anthropic 旗下 **Anthropic Labs** 近日推出了名为 **Claude Design** 的实验性新产品，旨在让用户通过与 **Claude** 对话来创建交互原型、幻灯片、单页文档等精美的视觉设计作品。

该产品由 **Anthropic** 目前最强大的视觉设计模型 `Claude Opus 4.7` 驱动，以研究预览版的形式向 **Claude Pro**、**Max**、**Team** 和 **Enterprise** 订阅用户逐步开放。

根据官方博客显示，用户只需描述需求，**Claude** 便会构建初版视觉内容，随后可通过对话、行内评论、直接编辑或自定义滑块进行微调。

在初始引导阶段，该产品能够读取团队的代码库和设计文件，自动构建专属的设计系统，从而确保后续生成的内容与企业的整体视觉风格保持一致。

此外，**Claude Design** 还内置了网页捕获、文档导入（DOCX、PPTX、XLSX）等丰富的输入方式，并支持将最终成果导出为 **Canva** 项目、PDF、PPTX、独立 HTML 文件，或直接打包成交接包无缝传递给 `Claude Code` 进行下一步开发。

## [xAI 上线 Grok 4.3 测试版，启用云计算机模式支持文件操作](https://x.com/testingcatalog/status/2045071630154535363) `#2`
> xAI 现向部分用户开启新模型 **Grok 4.3** 测试版的早期访问。据用户观察，该版本放弃了前代的 16 Agent 架构，转型为原生配备 `shell` 与持久化文件层的 Agent，能一键生成结构化的幻灯片、PDF 以及高完成度的 Web 前端页面。

**xAI** 旗下新一代模型 `Grok 4.3 (beta)` 现已面向部分用户开启 Early Access 测试，并在官方应用及网页端的模型选择菜单中上线。

据用户观察，与此前采用 16 Agent 架构的主力版本 `Grok 4.20` 不同，该版本转型为云 Computer 模式，原生配备了 shell 与持久化文件层。

测试表明，其支持直接从视频中提取音频文件，或深入分析视频片段提取核心论点，进而自动匹配视觉基调并生成结构化的 PowerPoint 幻灯片。

此外，该模型还能创建 PDF、Sheets、Docs 等多种格式的文件。

## [QuiverAI 发布 Arrow 1.1 SVG 生成模型](https://quiver.ai/blog/introducing-arrow-1-1) `#3`
> **QuiverAI** 发布了最新 SVG 生成模型 `Arrow 1.1` 及其高精度版 `Arrow 1.1 Max`。该系列模型能通过代码直接生成高精确性和高一致性的矢量图形。
>
> 相关模型价格较前代有大幅降低，目前已在官方平台开放公测。

**QuiverAI** 最近正式发布了其最新的 SVG 生成模型 `Arrow 1.1` 及其高精度版本 `Arrow 1.1 Max`。

该系列模型的核心理念是通过代码有意识地生成矢量图形，而非对像素进行简单的近似处理。

与上一代相比，`Arrow 1.1` 在生成效率、输出质量、结构细节把控以及指令遵循方面均实现了显著提升。其通过增加基础几何图形的使用比例，减少了对复杂路径的依赖，从而输出了更易于编辑和扩展的结构化 SVG。

根据官方博客公布的数据，其生成和矢量化单价分别降至 **$0.20** 美元和 **$0.15** 美元。

此外，针对需要极高精度的技术图表等场景同步推出的 `Arrow 1.1 Max`，在色彩还原与形状对齐上表现出更高的稳定性，但其运行时间与成本也相应增加。

目前，上述模型已在 **QuiverAI** 的官方平台开放公开测试。

## [Cloudflare 宣布 Email Service 公测 支持 AI Agent 集成](https://developers.cloudflare.com/email-service/) `#4`
> Cloudflare 宣布专为 AI Agent 打造的双向邮件通信基础设施 **Email Service** 正式开启公测。同时，官方还开源了具备人机协作功能的参考应用 **Agentic Inbox**，并全面兼容 `Claude Code` 等外部 Agent 快速接入。

**Cloudflare** 宣布其 **Email Service** 正式进入公开测试阶段。该服务将 **AI Agent** 作为核心目标，提供完整的双向邮件通信基础设施。

通过结合免费的 **Email Routing** 与现已公测的 **Email Sending**，开发者可以在 **Cloudflare** 平台内实现接收、处理并异步回复邮件的完整闭环。官方表示，系统会自动配置 **SPF**、**DKIM** 和 **DMARC** 记录以确保高送达率。

在 **Agent** 架构方面，其深度集成了 **Agents SDK** 的 `onEmail` hook，利用 **Durable Objects** 实现跨邮件的状态持久化，并支持基于地址的智能路由以及 **HMAC-SHA256** 安全回复路由。

为了进一步降低开发门槛，**Cloudflare** 还开源了具备完整邮件客户端和人机协作功能的 "**Agentic Inbox**" 参考应用。

根据官方文档显示，该功能目前包含在 **Workers Paid plan** 中。

## [Opera 推出 Browser Connector 功能，支持外部AI读取标签页](https://press.opera.com/2026/04/16/opera-announces-browser-connector/) `#6`
> Opera 为其 **Opera One** 和 **Opera GX** 浏览器推出一项名为 **Browser Connector** 的功能。该功能基于 `MCP`，允许用户将 **ChatGPT** 等外部 AI 直接接入浏览器实时会话，使 AI 能够读取标签页内容及截图分析。

Opera近日为其 **Opera One** 和 **Opera GX** 浏览器推出了一项名为“Browser Connector”的全新免费功能。该功能基于 `MCP` 标准，允许用户将 **ChatGPT** 和 **Claude** 等外部AI工具直接接入其浏览器的实时会话中，赋予这些AI获取完整上下文的能力。

通过这项技术，AI能够实时读取打开的标签页及页面内容，甚至通过截图来分析图像与图表，从而彻底免去了过去使用外部AI服务时需要人工频繁复制粘贴、充当“中间人”传递信息的繁琐步骤。

目前，该功能已在浏览器的 **Early Bird** 测试环境中上线。用户需在设置中开启 **Early Bird** 模式并重启浏览器，随后在“AI Services”选项中安装该扩展，登录 **Opera** 账号并按照指引完成与 **ChatGPT** 或 **Claude** 的授权连接即可体验。
