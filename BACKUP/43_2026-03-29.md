## [飞书开源 lark-cli，支持 AI Agent，覆盖 11 个业务域](https://github.com/larksuite/cli) `#2`
> 飞书发布了开源命令行工具 **“lark-cli”**。这款工具覆盖了飞书平台即时通讯、日历、文档等 **11** 个核心业务领域，内置 **19** 个结构化技能，兼容多种主流 AI 工具，`Agent` 无需额外配置即可直接操作飞书。

飞书官方在 GitHub 上开源了名为 `lark-cli` 的命令行工具。该工具专为人类和 AI Agent 设计，能够覆盖日历、即时通讯、文档、多维表格等 11 个核心业务领域。

`lark-cli` 具备原生的 Agent 设计理念，内置 19 个结构化的 AI Agent Skills，兼容主流 AI 工具，使得 Agent 无需额外配置即可操作 Lark。

其采用三层命令架构，不仅为人类提供了带有智能默认值和表格输出的 Shortcuts，还为高级用户和系统提供了映射 1:1 平台端点的 API Commands，以及覆盖 2500 多个 API 的 Raw API 调用能力。

## [RaBitQ 团队质疑 Google TurboQuant 论文存学术不端嫌疑](https://zhuanlan.zhihu.com/p/2020969476166808284?share_code=1qgRqvv3O7jrE&utm_psn=2021129597194962773) `#5`
> 苏黎世联邦理工学院博士后**高健扬**发布公开声明，指控谷歌被 **ICLR** 接收的 `TurboQuant` 论文存在多个问题：存在系统性地回避 `TurboQuant` 方法与已有 `RaBitQ` 方法的相似性，错误描述 `RaBitQ` 的理论结果，刻意创造不公平的实验环境。
>
> `RaBitQ` 团队已向 **ICLR** 大会主席及委员会提交正式投诉，并计划发布技术报告以澄清事实。

据苏黎世联邦理工学院博士后、**RaBitQ** 系列工作第一作者**高健扬**发布的公开声明指出，**Google Research** 近期被 **ICLR 2026** 会议接收的论文《**TurboQuant: Online Vector Quantization with Near-optimal Distortion Rate**》存在学术不端嫌疑。

该声明详细控诉了 **TurboQuant** 团队系统性地回避其方法与 **RaBitQ** 核心设计的高度相似性、在没有提供任何论据的情况下错误地将 **RaBitQ** 的理论结果定性为“次优”，以及在实验对比中刻意使用劣化实现和单核 CPU 环境测试 **RaBitQ**，却使用 **A100 GPU** 测试自身算法，从而创造不公平的对比条件。

尽管 **RaBitQ** 团队在此前已多次通过邮件明确指出这些事实性错误并要求修正，**TurboQuant** 第一作者仅承诺在会议结束后部分修正理论和实验问题，且拒绝讨论两者方法间的相似性。

目前，该争议论文已通过 **Google** 官方渠道获得数千万次社交媒体浏览量，**RaBitQ** 团队已向 **ICLR** 大会相关主席及委员会提交正式投诉，并计划在 **arXiv** 发布详细技术报告以澄清事实。
