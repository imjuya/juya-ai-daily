## [Claude Code 桌面版发布重大更新](https://x.com/claudeai/status/2024937960572104707) `#2`
> ``Claude Code`` 桌面版迎来重大更新，新增 ``Server Previews``、``Local Code Review``、``PR Monitoring`` 及 ``Session Mobility`` **四大**核心功能。``Claude`` 现在能直接在桌面端启动服务器预览应用，自动审查`代码`中的`Bug`，并监控`CI`状态，实现自动修复与合并。

**Claude** 官方宣布 `Claude Code` 桌面版迎来重大更新，新增 `Server previews`、`Local code review`、`PR monitoring` 及 `Session mobility` 核心功能。更新后，**Claude** 可在桌面端直接启动开发服务器并预览应用，结合日志与截图进行迭代；代码提交前支持行内 Bug 审查；PR 流程中具备 `Auto-fix` 和 `Auto-merge` 能力，可自动修复故障并在通过检查后合并代码。此外，`Session mobility` 实现了 `CLI` 与桌面、云端及移动端间的会话无缝迁移。

## [Anthropic 发布 Claude Code Security](https://www.anthropic.com/news/claude-code-security) `#3`
> **Anthropic** 推出了集成在 `Claude Code` 中的新功能 `Claude Code Security`，该功能由 `Claude Opus 4.6` 驱动，能像人类安全专家一样对代码进行推理，从而发现传统工具难以捕捉的业务逻辑漏洞。**Enterprise** 和 **Team** 客户以及开源项目维护者现可申请试用。

**Anthropic** 官方宣布推出 `Claude Code Security`，现处于有限研究预览阶段。该功能集成在 `Claude Code (Web)` 中，由 `Claude Opus 4.6` 驱动。不同于依赖`模式匹配`的传统`静态分析工具`，该工具能像人类一样推理代码，通过理解组件交互和数据流向，发现`业务逻辑缺陷`等`复杂漏洞`。官方数据显示，该模型已协助团队在开源代码中发现超过 **500** 个隐藏漏洞。所有发现均经过多阶段验证和评级，修复需经人工批准。目前，**Enterprise** 和 **Team** 客户及开源项目维护者均可申请访问。

## [Karpathy 称传统 App Store 模式将过时](https://x.com/karpathy/status/2024583544157458452) `#10`
> **Andrej Karpathy** 展示了他利用 `Claude` 仅用 **1 小时**就构建出的定制化心肺训练仪表盘，以此预言传统的 **App Store** 模式将过时，未来将由 `LLM Agent` 即时生成满足特定长尾需求的软件。

**Andrej Karpathy**通过构建定制化心肺训练追踪仪表盘，展示了他对未来“高度定制化软件”的设想。为执行一项为期**8周**的计划，旨在通过`Zone 2`训练与`HIIT`将静息心率从**50**降至**45**，他利用`Claude`在**1小时**内编写了一个约**300行代码**的`Web应用`。该应用通过`逆向工程`**Woodway**跑步机云端`API`获取原始数据，尽管过程中出现单位换算和日历匹配错误需人工修复，但他认为这相比于**2年前**完成同样工作所需的**10小时**，效率已大幅提升。

**Karpathy**指出，传统**App Store**模式正变得过时，因为针对高度特定的长尾需求，`LLM Agent`能够即时生成完全贴合需求的定制应用。他认为当前行业进展缓慢，**99%**的产品本质上作为传感器，却仍只提供人类可读的文档和`前端`，缺乏`AI原生`的`CLI`和`API`，迫使Agent进行`逆向工程`。他展望未来，行业应重构为具备`Agent原生工效学`的传感器与`执行器`服务，通过“`LLM胶水`”进行编排，并借助拥有大量个人上下文的`AI`，最终实现仅需**1分钟**即可构建高度定制、临时化应用程序的目标。
