## [月之暗面正式发布并开源了 Kimi K2.6 模型](https://www.kimi.com/blog/kimi-k2-6) `#1`
> 月之暗面正式发布并开源了 Kimi K2.6 模型。该模型采用混合专家架构，总参数量1T，激活参数32B，上下文长度支持 256K。官方称这款模型在代码编写、长程任务执行和Agent集群能力方面均达到行业领先水平，并在多项基准测试中表现优于多款主流闭源模型。目前该模型已通过官方渠道以及各大第三方平台全面上线。

**月之暗面** 近日正式发布并开源 `Kimi K2.6` 模型。

`Kimi K2.6` 采用拥有 `1T` 总参数与 `32B` 激活参数的混合专家架构，上下文长度支持 `256K`。

官方称，该模型在代码、长程任务执行和 `Agent` 集群能力上达到了行业领先水平，并在多项基准测试中表现持平或优于多款主流闭源模型。

目前，该模型已在 kimi.com、Kimi App、Kimi API 及 Kimi Code 以及各类第三方平台全面上线。

其核心能力体现在长程编码、代码驱动设计、`Agent` 集群架构升级以及主动式 `Agent` 的协同上。

它不仅能够不间断执行复杂的代码重构与优化任务，还能通过升级后的"`Agent` 集群”调度最多 `300` 个子 `Agent` 并行处理 `4000` 个协作步骤。

此外，官方已开启名为"**Claw 群组**"的小范围内测。

该内测旨在支持人类与异构 `Agent` 在同一群组内协同工作，并进一步整合了 Office 文档处理与技能创建功能，以降低专业工作流的门槛。

## [OpenAI 为 Codex 上线 Chronicle，支持屏幕上下文记忆功能](https://developers.openai.com/codex/memories/chronicle) `#3`
> **OpenAI**为 **Codex**推出了名为 `Chronicle`的实验性新功能，它能通过捕获用户的屏幕画面来自动生成记忆，省去重复沟通背景信息的麻烦。
>
> 不过官方也警告，这项功能不仅会快速消耗额度，还存在提示注入攻击以及本地明文存储带来的隐私风险。
>
> 目前该功能仅面向部分地区的 macOS 版 ChatGPT Pro 订阅用户开放。

**OpenAI** 为其编程工具 **Codex** 推出了一项名为 `Chronicle` 的实验性新功能。作为对上周发布的 `Memories` 功能的扩展，`Chronicle` 能够通过捕获和分析用户的近期屏幕上下文来构建记忆，从而减少用户在与 **Codex** 交互时重复陈述背景信息的需求。

该功能目前以选择性加入的研究预览版形式提供给 macOS 平台上的 **ChatGPT Pro** 订阅用户，暂不支持欧盟、英国和瑞士地区。要启用此功能，用户需在 **Codex** 应用的设置中开启 `Memories` 选项及 `Chronicle`，并授予 macOS 的屏幕录制和辅助功能权限。

官方表示，该技术通过在后台运行沙盒化的 `Agent` 来处理屏幕截图、OCR 文本和时间信息等内容，帮助 **Codex** 识别用户的工作流及所需工具。然而，官方也明确警告，启用 `Chronicle` 会迅速消耗速率限制，增加提示注入攻击的风险，且生成的记忆会以未加密的 Markdown 文件形式存储在本地设备上。

## [阿里发布 Fun-ASR1.5，支持 30 种语言及方言识别](https://mp.weixin.qq.com/s/xPEstUiJ4IC3BOFVsdvRLA) `#5`
> **阿里**发布了端到端语音识别大模型 `Fun-ASR1.5`。该模型无需预设语种就能自动切换并精准识别 **30 种语言**。
>
> 不仅攻克了多种中文方言和古诗词的识别难题，还能智能添加标点并将口语自动转换为规范文本。
>
> 目前已上线 **阿里云百炼**并在 **魔搭**提供体验。

**阿里**发布了端到端语音识别大模型 `Fun-ASR1.5`。该模型作为新一代版本，实现了单模型高精度识别 **30 种**语言，覆盖中文七大方言体系及二十余种地方口音，并强化了对中文古诗词诵读的专项识别能力。

官方表示，其在跨语言场景下无需预设语种标签即可自动识别切换，且基于数十万小时方言数据训练使平均字错误率相比上一版本下降了 **56.2%**。此外，该模型在后处理环节优化了智能标点预测与文本归一化（`ITN`）能力，使转写文本更加规范。

目前，用户已可通过**阿里云百炼平台**调用其 API，或在**魔搭社区**演示进行体验。

## [GitHub 调整 Copilot 订阅计划](https://github.blog/changelog/2026-04-20-changes-to-github-copilot-plans-for-individuals/) `#6`
> **GitHub**宣布调整 **Copilot** 个人版计划，目前已暂停部分付费版本的新用户注册并收紧了使用限额。
>
> 受影响的现有付费用户可以在公告发布后一个月内联系支持团队申请免责退回**4 月**费用。

**GitHub** 宣布对 **Copilot** 个人版计划进行调整，旨在保障现有付费用户的服务可靠性与提供可持续的体验。

目前，官方已暂停 **Copilot Student**、**Pro** 和 `Pro+` 计划的新用户注册。不过，**Copilot Free** 仍开放注册，且现有用户依然可以在上述各计划间进行升级。

在额度方面，官方收紧了个人版计划的使用限额，其中 `Pro+` 计划的限额在 **Pro** 的 **5 倍**以上。

此外，`Opus` 模型已被从 **Pro** 计划中移除。该模型中的 `Opus 4.7` 版本目前仅在 `Pro+` 计划中继续可用。

针对受影响的群体，官方出台了免责退费政策。若用户遇到意料之外的限额或认为变更不适用，可取消 **Pro** 或 `Pro+` 订阅且不会被收取 **4 月**费用。用户需在官方指定的退费申请期联系支持团队获取退款。

**ZCode** 日前正式发布了全新的 **1.0.0** 版本。该版本基于全新 `Electron` 架构构建，桌面端体验、流畅度与稳定性均得到显著提升。
