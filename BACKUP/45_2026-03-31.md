## [阿里千问发布 Qwen3.5-Omni 系列模型](https://qwen.ai/blog?id=qwen3.5-omni) `#2`
> 阿里千问正式发布了新一代全模态大模型 **Qwen3.5-Omni** 系列，包含 **Plus**、**Flash** 和 **Light** 三种尺寸。
>
> 该模型原生实现了对文本、图片、音频及音视频的统一理解，同时支持超长音视频输入、音色克隆、智能语义打断以及 **113** 种语种和方言的语音交互。
>
> 目前，企业及开发者已可通过 **Qwen Chat**、阿里云百炼 API 进行体验和集成。

**Qwen Team** 近日正式发布了新一代全模态大模型 **Qwen3.5-Omni** 系列，包含 **Plus**、**Flash**、**Light** 三种尺寸。该模型原生支持文本、图片、音频及音视频的理解，其 **Thinker** 与 **Talker** 模块均升级为 `Hybrid-Attention MoE` 架构，支持 256k 长上下文，单轮可处理最长 3 小时的音频输入或 1 小时的 720P 音视频输入。

根据官方提供的数据，**Qwen3.5-Omni-Plus** 在 215 项子任务和 Benchmark 上取得 SOTA 成绩，其通用音频能力全面超越 **Gemini-3.1 Pro**，音视频理解能力总体达到 **Gemini-3.1 Pro** 水平，视觉与文本能力则与同尺寸的 **Qwen3.5** 模型效果持平。

新模型涌现出 **Audio-Visual Vibe Coding** 能力，可根据音视频指令直接生成代码，并具备剧本级音视频 Caption 生成、音色克隆、智能语义打断及原生 WebSearch 调用等特性，支持 113 种语种和方言的语音识别及 36 种语种和方言的语音生成。

目前，用户可通过 **Qwen Chat**、阿里云百炼 API 以及 **Hugging Face** 和 **ModelScope** 的 Demo 进行体验。

## [美团龙猫团队发布并开源 LongCat-AudioDiT 模型](https://github.com/meituan-longcat/LongCat-AudioDiT) `#3`
> 美团龙猫团队推出了文本转语音模型 `LongCat-AudioDiT`，提供 **1B** 和 **3.5B** 两种规格。该模型直接在波形潜空间运行，支持高保真的中英双语音频生成。

美团龙猫团队发布非自回归扩散 TTS 模型 **LongCat-AudioDiT**。其在波形潜空间运行，仅需 `Wav-VAE` 和扩散主干，规避中间声学表示依赖。

官方称引入自适应投影引导（APG）取代传统 CFG，修复训练与推理不匹配问题。

该模型提供 **1B** 和 **3.5B** 两种规格，支持中英高保真生成。

官方数据显示，**3.5B** 版本在 **Seed** 基准测试中实现 SOTA 零样本语音克隆性能，SIM 得分在 **Seed-ZH** 从 **0.809** 升至 **0.818**，**Seed-Hard** 从 **0.776** 升至 **0.797**。

目前，项目代码及权重已在 GitHub 和 Hugging Face 开源。

## [微软发布 Harrier-OSS-v1 多语言嵌入模型](https://huggingface.co/microsoft/harrier-oss-v1-27b) `#5`
> 微软发布了 **Harrier-OSS-v1** 多语言文本嵌入模型系列，涵盖 **270M** 到 **27B** 三种参数规模，全面采用 `Decoder-only` 架构并支持 **32k** 上下文窗口，突破了传统模型处理长文本的瓶颈。

微软近日发布了名为 `Harrier-OSS-v1` 的多语言文本嵌入模型系列。该系列包含 **270M**、**0.6B** 和 **27B** 三种参数规模，全面采用与现代大语言模型相似的 `Decoder-only` 架构，并结合 `last-token pooling` 与 `L2 normalization` 技术来生成密集的文本向量。

所有上述模型均统一支持高达 **32,768** 个 token 的上下文窗口。该系列模型可广泛应用于检索、聚类、语义相似度计算、分类、双语文本挖掘及重排等多种任务。

开发者目前可通过 Hugging Face 获取该系列模型的权重，在使用时需特别注意其为指令微调模型，必须在查询端前置单句任务描述指令，而文档端则无需添加任何指令。
