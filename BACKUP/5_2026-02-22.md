## [智谱为 GLM Coding Plan 规则变动与体验下降致歉并提出补偿方案](https://mp.weixin.qq.com/s/qgVsa17L5Seqw9yXr1iExQ) `#1`
> **智谱**团队就 `GLM Coding Plan` 规则变动与体验下降致歉，支持受影响用户申请退款，留存用户获赠**15**天时长。还允许在**2月12日至16日**期间误升级的用户，权益升级至无周限额的老套餐，并承诺未来核心权益调整将提前说明。

**智谱团队**就 `GLM Coding Plan` 规则变动与体验下降向用户致歉，承认在 `规则透明度`、`灰度节奏` 及 `老用户升级机制` 上存在失误。

官方说明 `GLM-5` 参数规模达 `GLM-4.7` **两倍**以上，采取高峰期**3倍**、非高峰期**2倍**的消耗策略，看板刷新频率已优化至**10分钟**。因流量激增及遭受攻击，服务按 `Max`、`Pro`、`Lite` 顺序开放，目前 `Max` 已全面开放，`Pro` 高峰期限流，`Lite` 将在之后 `灰度开放`。

解决方案包括：支持 `Lite`/`Pro` 用户退款；赠送已使用及留存用户**15天**时长；**2月12日至16日**误升级用户可一键升级至 `无周限额老套餐`。官方承诺未来 `核心权益调整` 将提前说明。

## [Claude Code 引入内置 Git Worktree 支持](https://t.co/cjOXhUDL0i) `#3`
> **Claude Code CLI** 上线了内置 `Git Worktree` 支持，将此前仅限 **Desktop** 端的隔离能力扩展至 `CLI`，解决多个 `Agent` 并行运行时的文件冲突问题。

**Claude Code** 官方在 **2.1.50** 版本中引入了内置 `Git Worktree` 支持，将此前仅限 `Desktop` 应用的功能扩展至 `CLI`。此功能通过 `Git Worktree` 机制，允许多个 `Agent` 在同一仓库的独立工作目录中并行运行，解决了因共享工作目录导致的文件冲突与覆盖问题。

目前，`CLI` 端支持 `claude --worktree` 参数启动，可自动或手动命名工作树；`Desktop` 应用增设了 `Worktree` 模式开关；`Subagent` 同样支持 `Worktree` 隔离，便于执行大规模代码迁移等并行任务；用户可通过 `Frontmatter` 配置 `"isolation": "worktree"`，使自定义 `Agent` 默认在隔离环境中运行；该设计还兼容 `Mercurial`、`Perforce` 等非 `Git` 版本控制系统，用户可通过定义 `Hooks` 实现同等隔离效果。

## [Andrej Karpathy 评析 Claw 技术及安全隐患](https://x.com/karpathy/status/2024987174077432126) `#8`
> **Andrej Karpathy** 近期专门购置 **Mac Mini** 测试 `Claws` 技术，他将 `Claws` 定义为位于 `LLM Agent` 之上的新层级，负责任务的编排与调度。

**Andrej Karpathy** 近期购买 `Mac Mini` 以测试 `Claws` 技术。**Karpathy** 将 `Claws` 定义为位于 `LLM Agent` 之上的新层级，负责编排、调度与持久化。

尽管看好该概念，**Karpathy** 对主流实现 `OpenClaw` 持保留态度。他称 `OpenClaw` 是一个由 `vibe coding` 构成的 **40万行代码** 庞然大物，存在 `RCE（远程代码执行）` 漏洞和供应链投毒等严重安全隐患，现状堪比“安全噩梦”。

他更看好 `NanoClaw` 等轻量级方案。`NanoClaw` 核心引擎仅约 **4000行代码**，且默认在 `容器` 中运行。其架构特点是通过 `Skills 指令` 修改源码来集成功能，而非依赖 `配置文件`，**Karpathy** 赞赏其为一种防止配置混乱的新范式。与此同时，社区也涌现出 `nanobot`、`nullclaw` 等多个同类项目。

## [据报 OpenAI 文件上调了五年收入预期](https://x.com/theinformation/status/2025280618917945476) `#10`
> 据 **The Information** 报道，**OpenAI** 大幅上调五年期收入预测，预计今年营收达 **300 亿美元**，目标是 **2030 年**实现 **1500 亿美元**的消费者销售。`ChatGPT` 周活用户已升至 **9.1 亿**，但高昂的 `算力成本` 导致毛利率下滑，公司预计到 **2030 年**在基础设施上的投入将高达 **6650 亿美元**，并计划在当年实现 `现金流转正`。

据媒体 **The Information** 报道，**OpenAI** 在内部文件中大幅上调**五年期**收入预测约 **27%**，但预计到 **2030 年**现金消耗将高达 **1110 亿至 1120 亿美元**。

文件显示，公司去年收入达 **131 亿美元**，今年和明年的收入目标分别为 **300 亿**和 **620 亿美元**。其中，面向消费者的销售预计今年翻倍至 **170 亿美元**，并计划在 **2030 年**达到 **1500 亿美元**。

高额的 `算力` 成本是主要支出。公司预计到 **2030 年**在基础设施和计算资源上的投入将达 **6650 亿美元**，导致 `调整后毛利率` 从 **40%** 降至 **33%**，未达预期目标。今年的总支出预计为 **250 亿美元**，明年将增至 **570 亿美元**。

用户增长方面，**ChatGPT** `周活跃用户`数已达 **9.1 亿**，受益于 `GPT-5.1` 和 `5.2` 等更新带来的体验提升，公司长期目标是在 **2030 年**实现 **27.5 亿**`周活跃用户`。尽管面临巨额投入，**OpenAI** 仍预计在 **2030 年**实现 `现金流转正`，截至去年年底其账面现金约为 **400 亿美元**。
