## [Codex 发布 Codex Security 插件，集成五个安全工作流](https://x.com/reach_vb/status/2051019108028969251) `#3`
> **Codex**推出了**Codex Security**安全扫描插件，内置安全扫描、威胁建模、漏洞发现、验证和攻击路径分析这五个标准化的`AppSec`工作流。

**Codex** 团队发布了“**Codex Security**”安全插件，目前已可使用。

该插件将安全审查整合为一条自动化流水线，内置五个应用安全工作流：

`安全扫描` 对 PR、分支及代码仓库进行全面扫描。

`威胁建模` 自动绘制仓库地图，识别资产及潜在失效模式。

`漏洞发现` 探查权限绕过、注入攻击等多种潜在漏洞。

`验证` 通过构建概念验证和执行测试等方式，分离有效信号与误报。

`攻击路径分析` 将核实发现转化为攻击者视角的报告，评估严重性并生成最终报告。

## [OpenClaw 发布 2026.5.2 版本，重写插件架构优化运行性能](https://github.com/openclaw/openclaw/releases/tag/v2026.5.2) `#4`
> **OpenClaw** 发布了 **2026.5.2** 版本，通过重写 `插件架构`实现`npm`优先安装，大幅优化了`Gateway`与`Agent`的启动性能。
>
> 新版本接入了 **Grok 4.3** 支持，并集中修复了主流消息通道的边缘问题，提升了整体稳定性。

近日，**OpenClaw** 官方正式发布 `2026.5.2` 版本。

该版本重写了插件架构，全面切换至 `npm` 优先安装路径以解决此前的依赖问题，并新增 Beta 通道回退及医生修复功能。

性能上，显著优化了 `Gateway` 与 `Agent` 的启动及热路径，有效降低了就绪延迟与重复 `CPU` 工作。

同时，大量修复了 `Discord`、`Slack` 及 `WhatsApp` 等消息通道的边缘情况。

此外，新增 `xAI Grok 4.3` 为默认模型，并完善了多项网络搜索引擎集成。

官方表示，此版本解决了近期升级中用户遇到的多数问题。

## [网易内测 ClawEmail，为 AI Agent 分配真实邮箱](https://claw.163.com/) `#5`
> **网易**正在内测一款名为 **ClawEmail** 的产品，可以为每个 `Agent` 分配真实的邮箱地址。
>
> 只要向该邮箱发信就能触发 `Agent` 处理任务并自动收到回复。
>
> 目前处于内测阶段。

**网易 163**团队推出邮件处理引擎`ClawEmail`，定位"`Email for AI Agent`"。

该产品内测期间免费，每账号限建**5**个 Agent 邮箱，每月发信额度**3000**封。用户可为 Agent 分配专属`@claw.163.com`邮箱，发送邮件即可调用 Agent 并自动获回复。

产品支持主流框架集成，具备邮件收发、正文解析、附件处理、检索及批量管理功能，单附件限**30MB**。
