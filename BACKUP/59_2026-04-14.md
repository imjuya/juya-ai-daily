## [Anthropic 回应提示词缓存时长争议](https://x.com/bcherny/status/2043715713551212834) `#10`
> 针对开发者质疑 **Claude Code** 在关闭遥测后，提示词缓存时长会从一小时降至五分钟的争议，官方工程师澄清，这并非牺牲隐私换性能的惩罚机制，而是客户端实验开关失效导致读取了默认值。
>
> 并称未来更新中将提供环境变量，允许所有用户根据实际需求手动强制切换缓存时长。

近日，开发者 **Can Vardar** 发现 **Anthropic** 旗下产品 `Claude Code` 在用户关闭遥测功能后，其 prompt cache 时长会从 **1 小时** 缩短至 **5 分钟**，并据此推测此举相当于让用户以牺牲隐私换取 **12 倍** 的性能差异。

对此，**Anthropic** 工程师 **Boris Cherny** 出面澄清，表示这并非官方故意惩罚隐私保护行为，而是由于遥测关闭导致客户端的实验开关失效，系统从而读取了 **5 分钟** 的默认值。

该工程师强调，**1 小时** 的 prompt cache 并非在所有场景下都具备优势，其写入成本更高而读取成本更低，实际效果高度依赖上下文窗口大小及查询属于主 Agent 还是 subagent 等使用模式。

目前，该公司已在部分场景中为订阅用户默认推出 **1 小时** 缓存以优化延迟和 Token 消耗，但尚未对 API 客户默认开启。

其团队计划在后续更新中将部分查询的客户端默认值调整为 **1 小时**，并提供环境变量允许用户手动强制切换 **1 小时** 或 **5 分钟** 的缓存时长。

同时，官方明确驳斥了 **12 倍** 性能差距的说法，指出实际节省的 Token 量远低于此。

## [Cursor 回应内置 Claude Agent SDK 争议](https://x.com/dotey/status/2043588734952563180) `#11`
> 有开发者逆向发现，**Cursor 3.0** 不仅内置了完整的 `Claude Agent SDK`，还将相关标识全面替换为 **Cursor**。
>
> **Cursor** 创始人对此澄清，这是在不到 **1%** 流量上进行的 A/B 测试，目的是对比 `Claude` 框架与 `Cursor` 框架的表现差异。

近日，有开发者对 **Cursor** 3.0 版本进行逆向工程后发现，其 `Cursor Agent` 实际上打包了完整的 **Anthropic** 官方 `Claude Agent SDK` 和相关 CLI 工具。该软件会在本地启动一个 HTTP Proxy，通过实时字符串替换引擎将“Claude”相关字眼（如将 `Claude Code` 替换为 `Cursor Agent`，`CLAUDE.md` 替换为 `AGENTS.md`）全面替换为 **Cursor** 标识，并剥离 **Anthropic** 的计费头和 sandbox 引用等信息。

针对“套壳”质疑，**Cursor** 创始人 **Michael Truell** 出面澄清，指出这只是该团队进行的一项不到 1% 流量的小范围 A/B 测试，旨在对比“原生 `Claude Code` harness”与“**Cursor** 默认 harness”的表现差异，属于团队持续优化 Agent 速度、稳定性和准确性的常规离线评估与测试手段。
