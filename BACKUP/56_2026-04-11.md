## [Anuttacon 发布 LPM 1.0 模型，仅供学术交流](https://large-performance-model.github.io/) `#3`
> 由米哈游创始人**蔡浩宇**创立的AI公司**Anuttacon**正式公布了自研视频角色表演生成模型`LPM 1.0`。该模型采用**17B**参数扩散架构，支持多模态导演级控制与零样本泛化，可作为虚拟主播和游戏NPC等场景的实时视觉引擎。
>
> 官方称，该项目现阶段仅用于学术交流，无任何开源计划，也不提供任何API与产品服务。

近日，视频角色表演生成模型 **LPM 1.0** 的研究论文及项目主页正式公布。该模型由米哈游创始人**蔡浩宇**创立的 AI 公司 **Anuttacon** 自研。

根据官方论文与项目主页显示，该模型专注于单人全双工音视频对话表演，采用 **17B** 参数量的 `Diffusion Transformer` 架构，并蒸馏出因果流式生成器以实现低延迟、无限时长的实时交互。

**LPM 1.0** 旨在解决生成高表现力、实时推理与长期身份稳定之间的“表演三难困境”。其通过多模态输入（图像、音频、文本）实现了对角色说话、聆听及演唱等细腻表演的导演级控制，并具备对写实真人、2D/3D 及非人形等任意风格角色的零样本泛化能力。

官方称，该模型在其提出的首个交互式角色表演基准 **LPM-Bench** 的所有评估维度上均达到了 SOTA 水平，可作为对话代理、虚拟主播和游戏 NPC 的视觉引擎。

然而，官方同时明确声明，该项目目前仅用于学术交流，尚无计划向公众开源模型权重、源代码或提供任何 API 及产品服务。

## [Claude Code 推出 ultraplan 支持云端规划本地执行](https://code.claude.com/docs/en/ultraplan) `#4`
> Claude Code 推出了名为 `ultraplan` 的研究预览版新功能，将代码分析与方案起草转移至云端处理，以释放本地资源。
>
> 用户在本地终端输入指令发起规划后，可直接在浏览器中对云端生成的方案进行批注和迭代审阅。
>
> 确认后，既能在云端执行并创建 PR，也可传回本地继续实施。

**Claude Code** 推出了一项名为 **ultraplan** 的新功能。该功能允许用户在本地 CLI 中发起规划任务，并将代码库分析与方案起草工作交由 **Anthropic** 云端处理，从而释放本地终端资源。

在云端生成方案后，用户可在浏览器内对其进行段落批注、表情反馈及基于大纲的审阅迭代。方案确认后，用户既可选择在云端环境中直接执行并创建 pull request，也可将其“传送”回本地终端继续实施。

该功能目前处于研究预览阶段，要求使用 **Claude Code** `v2.1.91` 或更高版本，且需配置 **Claude Code on the web** 账户及 **GitHub** 仓库。

用户可通过 `/ultraplan` 命令、提示词包含关键词或在本地计划审批对话框选择发送三种方式启动。

任务提交后，终端显示状态指示器，用户可运行 `/tasks` 查看详情或停止。

由于运行依赖于 **Anthropic** 的云基础设施，该功能不支持 **Amazon Bedrock**、**Google Cloud Vertex AI** 或 **Microsoft Foundry** 环境。

## [Qwen Code 更新，支持远程控制及定时任务](https://x.com/Alibaba_Qwen/status/2042551216769765449) `#5`
> **Qwen Code** 发布更新，开发者现在能通过**钉钉**或**微信**等社交平台用手机远程控制执行代码，并可利用定时任务功能让**AI**自动运行循环测试。
>
> 同时，新版支持在执行前进行全局规划，并为不同子任务灵活分配模型。

Qwen Code 的 **v0.14.0** 至 **v0.14.2** 版本现已全面上线。

此次更新引入了多项重磅功能：用户不仅可以通过 **DingTalk** 或 **WeChat** 等平台远程控制该工具，实现“手机发消息、服务器收结果”的联动。

还能借助新增的 `Cron Jobs` 功能，安排如每 **30 分钟**自动运行测试或每日清晨定时拉取构建等循环 AI 任务。

在任务调度与执行层面，新版支持通过 `/plan` 命令进入“先规划后执行”模式，并允许用户为不同 `Sub-agent` 灵活分配模型，以兼顾性能与消耗。

此外，该工具还增加了任务完成后的智能后续建议、默认 **8K** 且截断时自动扩容至 **64K** 的 `Adaptive Output Tokens`。

以及通过 `Ctrl+O` 快捷键切换输出详细程度等高度实用的交互特性。

## [Anthropic 推出 Claude for Word 插件测试版](https://claude.com/claude-for-word) `#6`
> **Anthropic** 推出了 **Claude for Word** 插件测试版，该插件能以原生修订模式提供保留原格式的修改建议，还能自动总结修改记录并标记文档错误。目前仅向 **Claude** 的 Team 和 Enterprise 付费用户开放。

**Anthropic** 官方正式推出了 **Claude for Word** 插件的测试版。该插件使 **Claude** 能够直接在 Word 文档的侧边栏内运行，该工具即会以 Word 原生的“修订模式”呈现修改建议，供用户在审阅面板中接受或拒绝。

在处理文档时，该插件能智能继承原有的标题、编号和项目符号等格式，支持基于模板起草内容并引用上传的参考文档。它还具备完善的审阅辅助功能，包括处理评论互动、总结文档修订记录，以及自动标记不一致的术语和编号错误。

目前，上述功能仅面向 **Claude Team** 和 **Enterprise** 计划的付费用户开放，官方提醒用户在接受修改建议前务必进行人工审查。

## [美财政部联储召集银行，研讨 Anthropic 模型安全风险](https://news.bloomberglaw.com/ip-law/bessent-urgently-summons-bank-ceos-to-discuss-anthropics-new-ai) `#10`
> 据彭博社报道，美国财政部部长与美联储主席在华盛顿紧急召集**花旗**和**高盛**等多家银行CEO举行会议，重点讨论防范`Anthropic Mythos`模型可能带来的网络攻击风险。
