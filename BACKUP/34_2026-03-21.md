## Cursor 承认 Composer 2 基于 Kimi K2.5，月之暗面确认授权合作 `#1`
> Cursor 近日发布了新模型 `Composer 2`，但因底层被指基于 Kimi K2.5 且未注明而引发开源合规争议。
>
> Cursor 团队随后承认该模型确实以 Kimi K2.5 为基座，但强调在此基础上投入了数倍计算量进行持续预训练与强化学习，未及时披露属于失误。
>
> 最终 Kimi 官方确认双方存在通过 `Fireworks AI` 达成的授权商业合作。

Cursor 近日发布了新模型 **Composer 2**，但有开发者在调试 Cursor 的 **Composer 2** 模型时，发现请求体中的模型 ID 显示为 `kimi-k2p5-rl-0317-s515-fast`，即该模型是 **Kimi** k2.5 强化学习而来。

月之暗面员工最初对此表示两个模型的 Tokenizer 完全一致，质疑 Cursor 未遵守许可证注明模型来源。

对此，**Cursor** 官方随后承认 **Composer 2** 确实以 **Kimi** K2.5 为开源基座，但强调该团队在此基础上进行了大规模的持续预训练与强化学习，投入的计算量约为基座模型的两倍，且通过推理合作伙伴 **Fireworks** 的条款遵守了许可证规定。

**Kimi** 官方账号最终也确认了与 **Cursor** 的授权商业合作关系，并表示支持其模型集成工作。

## [Claude Code 上线云端定时，桌面版支持直接选 DOM](https://x.com/lydiahallie/status/2035126942366867785) `#2`
> **Claude Code** 针对网页端和桌面端推出了新功能。
>
> 网页端现已支持云端定时任务，用户可自行设定代码仓库、时间表和提示词。
>
> 桌面端则新增了直接选择 DOM 元素的交互方式，**Claude** 能自动获取元素的代码上下文和截图。

**Claude Code** 近期针对网页版和桌面版分别推出了新功能。

在网页版方面，该产品引入了云端定时任务功能，用户只需设置代码仓库、时间表和提示词，**Claude** 便会通过云端基础设施按计划自动运行。这意味着用户无需在本地维持客户端运行，甚至可以关闭浏览器标签页。

在桌面版方面，**Claude Code** 新增了直接选择 DOM 元素的交互方式。相较于以往通过文字描述指定更新组件，该功能允许 **Claude** 直接获取元素的标签、类名、关键样式、周围 HTML 代码以及裁剪后的截图。对于 React 应用，该功能还能进一步识别并捕获源文件、组件名称和属性。

## [美团龙猫开源 LongCat-Flash-Prover](https://github.com/meituan-longcat/LongCat-Flash-Prover) `#4`
> 美团龙猫团队发布了拥有 **5600亿** 总参数的开源 `MoE` 模型 `LongCat-Flash-Prover`，该模型专注于 `Lean4` 原生形式推理，在自动形式化和定理证明任务上有优异表现。

美团龙猫团队近日发布了 **LongCat-Flash-Prover**，这是一个拥有 **5600 亿参数**的旗舰级开源 **Mixture-of-Experts (MoE)** 模型。  

该模型致力于通过 Agent 式工具集成推理（TIR），推进 `Lean4` 中的原生形式推理能力，将复杂的推理任务分解为自动形式化、草图绘制和证明三个独立环节。  

官方数据显示，**LongCat-Flash-Prover** 在 **MiniF2F-Test** 上以每题 **72 次推理**实现了 **97.1%** 的通过率，并在 **PutnamBench** 上解决了 **41.5%** 的问题，确立了开源权重模型在自动形式化和定理证明任务上的新 SOTA 表现。
