## [智谱向 GLM Coding Plan 用户开放 GLM-5.1 模型使用](https://mp.weixin.qq.com/s/5g5-cJSuQumzZDVgiCaTuQ) `#1`
> 智谱向 **GLM Coding Plan** 全体用户开放了未正式发布的 **GLM-5.1** 模型的使用，该模型定位为能处理长周期任务的新一代编程 Agent。根据官方发布的基准测试数据，该模型较前代 **GLM-5** 模型有显著提升。
>
> 用户需在 Coding Agent 配置中手动将模型名称更改为 `glm-5.1` 方可接入使用。
>
> 官方称计划于 **4月上旬** 正式发布该模型。

智谱向 **GLM Coding Plan** 全部层级（Lite/Pro/Max）用户超前开放了尚未正式发布的 `glm-5.1` 模型。用户需在所使用的 Coding Agent 自定义配置中手动将模型名称更改为“glm-5.1”进行接入。

该模型作为高阶模型采用差异化额度消耗机制，高峰期（每日 **14:00-18:00 UTC+8**）按 **3倍** 系数抵扣，非高峰期常规按 **2倍** 抵扣。但作为限时福利，非高峰期抵扣系数降至 **1倍** 并持续至 **4月底**。

官方提示，在高负载时可能触发保护性限流，并将优先保障编程任务请求。**OpenClaw** 等非核心编程场景在集群繁忙时更易遇到限流。

## [Claude Code 上线 Windows PowerShell 原生运行支持](https://code.claude.com/docs/en/tools-reference#powershell-tool) `#4`
> **Claude Code** 针对 Windows 平台推出了更新，支持原生运行 `PowerShell` 命令以替代原有的 `Git Bash` 路由。目前该功能暂不支持自动模式与沙盒机制。

**Claude Code** 近期面向 Windows 平台推出了一项需主动启用的预览版 `PowerShell` 工具，允许其原生运行 `PowerShell` 命令而无需经由 `Git Bash` 路由。

用户可通过配置环境变量或 `settings.json` 文件来启用该功能。启用后，系统会自动检测 `pwsh.exe` (`PowerShell` 7+) 并向下回退兼容 `powershell.exe` (`PowerShell` 5.1)。

同时，官方提供了在交互命令默认设置、命令挂钩及技能前言中指定使用 `PowerShell` 的相关配置项。

由于处于预览阶段，该工具目前暂不支持 `Auto` 模式、配置文件加载、沙盒机制以及 `WSL` 环境，且启动 **Claude Code** 仍需依赖 `Git Bash`。

## [硅基流动推出 AI 算力运营服务](https://mp.weixin.qq.com/s/gW0g453oNAS4HO8U4k1VVg) `#9`
> 硅基流动推出 **AI算力运营服务**，帮助企业解决算力闲置与运维成本高的痛点。企业可通过联合运营模式按量分成，或在满足安全合规的前提下，利用算力消纳模式将自有冗余算力直接转化为额外收益。

硅基流动宣布推出 **“AI 算力运营服务”**，旨在帮助企业将现有算力转化为高效 Token 工厂，解决闲置与运维成本痛点。该服务支持 **英伟达**、**昇腾** 等多架构芯片统一接入与异构弹性调度。

合作模式分 **“联合运营”** 与 **“算力消纳”** 两种：前者面向 IDC 及芯片厂商，按服务量分成且试运行免服务费；后者面向政企及互联网企业，助其将冗余算力合规转化为收益。

官方确认该模型为 Claude Opus 之上的全新第四档，在编码及网络安全测试中得分大幅超越 `Claude Opus 4.6`。
