>

**Vercel** 近期开源代码安全扫描框架 `deepsec`。

## [Google 为 Gemini API 引入事件驱动 Webhooks 推送通知功能](https://blog.google/innovation-and-ai/technology/developers-tools/event-driven-webhooks) `#7`
> **Gemini API** 引入了 **Event-Driven Webhooks** 功能。
>
> 面对 **Agent** 工作流等长耗时任务，**API** 会在完成时主动向服务器推送 **HTTP POST** 通知，告别低效轮询。

**Google** 近期为 **Gemini** API 正式引入"**Event-Driven Webhooks**"功能，以推送式通知取代开发者处理耗时任务时的低效轮询操作。

任务完成时，API 会即时向指定服务器发送 `HTTP POST` 负载。该服务遵循 **Standard Webhooks** 规范，通过特定签名与时间戳保障操作幂等性并防重放攻击。

系统提供“至少一次”送达保证及 `24小时` 自动重试。开发者可进行 **HMAC** 全局配置或 **JWKS** 单次动态路由。

目前该功能已向所有开发者全面开放，官方同步上线了技术文档与编程指南。

## [Claude API平台推出 Workload Identity Federation功能](https://platform.claude.com/docs/en/build-with-claude/workload-identity-federation) `#8`
> **Anthropic** 旗下的 **Claude API** 平台推出了 `Workload Identity Federation` 功能。
>
> 开发者可利用 **AWS**、**GCP** 或 **GitHub Actions** 等环境的短时效 JWT 令牌进行无密钥认证调用 API。

**Anthropic** 为 **Claude** 平台正式推出了**工作负载身份联合**（`Workload Identity Federation`）功能，为开发者提供了不使用静态 API 密钥的“无密钥”认证方式。

该功能允许用户在 **Claude** 控制台中配置**身份提供商**（`IdP`）发放的 `JWT` 令牌与平台内服务账户之间的信任关系。

随后运行在 **AWS IAM**、**Google Cloud**、**GitHub Actions**、**Kubernetes**、**Microsoft Entra ID** 或 **Okta** 等环境中的工作负载即可用这些短时效令牌换取 **Anthropic** 访问令牌。

此举旨在消除创建、存储、轮换或泄露长期 API 密钥的风险。

## [OpenClaw 发布 2026.5.3 版本，新增文件传输与实时控制功能](https://github.com/openclaw/openclaw/releases/tag/v2026.5.3) `#9`
> **OpenClaw** 发布了 **2026.5.3** 版本。
>
> 新版本支持配对节点间文件传输，同时引入了 `/steer` 命令与 `/side` 命令，分别能实时干预任务以及进行不污染上下文的快速提问。
>
> 此外，插件体系全面转向官方 `npm` 包，系统性能与稳定性也得到显著提升。

**OpenClaw** 正式发布 **2026.5.3** 版本。该版本支持在已配对节点间传输文件，每个跃点上限 **16MB**，默认拒绝。
